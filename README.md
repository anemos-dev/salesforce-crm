# Asteria CRM

Salesforce風の社内CRMワークスペースです。営業、取引先、商談、ケース、キャンペーン、レポート、Agentforce風の推奨アクションを1つの画面で扱えます。

## 起動

```bash
npm install
npm run dev
```

`npm run dev` は以下を同時に起動します。

- React/Vite app: http://127.0.0.1:5173/
- CRM API: http://127.0.0.1:8787/

## 公開URL

GitHub Pages:

https://anemos-dev.github.io/salesforce-crm/

GitHub Pages版は静的サイトとして動作します。APIサーバーは起動しないため、PINログイン後は端末内の `localStorage` に保存されます。共有保存や競合制御を含めた確認はローカルの `npm run dev` で行ってください。

## データ保存

共有データは `data/workspace.json` に保存されます。画面上で商談、ケース、ステージ、確度などを更新すると、API経由でこのファイルに反映されます。

保存前のデータは `data/backups/` に自動退避されます。画面右上の `バックアップ` からJSONを手元に出力でき、`復元` から戻せます。

共有データには `_meta.version` が付いています。保存時に画面が保持しているバージョンとサーバー上の最新バージョンを照合し、他のユーザーが先に更新していた場合は `409` を返して上書きを止めます。画面には `再読込必要` と表示され、`共有データを再読込` から最新状態に戻せます。

APIに接続できない場合、画面は端末内の `localStorage` に退避保存します。ヘッダーに `共有保存` または `ローカル保存` と表示されます。

## ログインと権限

開発用の簡易PINログインを備えています。本番SSOやパスワード管理の代替ではありませんが、APIの読み書き境界を確認できます。

| ユーザー | PIN | 権限 |
| --- | --- | --- |
| 管理者 | `0000` | 読込/書込 |
| 営業マネージャー | `1111` | 読込/書込 |
| サポート担当 | `2222` | 読込/書込 |
| 閲覧者 | `3333` | 読込のみ |

APIは `/api/workspace` の読込/書込にトークンを要求します。閲覧者が書き込むと `403` になります。

## 検証

```bash
npm run lint
npm run build
```

APIだけ確認する場合:

```bash
npm run api
curl http://127.0.0.1:8787/api/health
curl http://127.0.0.1:8787/api/workspace
```

## 現在の運用レベル

この版は、社内PoCから小規模チームの試験運用に近い段階です。

できること:

- 左ナビによる業務ビュー切替
- 商談/ケースの新規作成
- 商談ステージ、確度、ケース優先度、対応状況の更新
- グローバル検索
- CSV出力
- JSONバックアップ/復元
- API経由のファイル永続化
- 更新前データの自動バックアップ
- 複数人同時編集の競合検知
- 簡易ログイン
- 閲覧者の書き込み拒否

本番運用前に追加すべきこと:

- 本番SSO/パスワード管理
- ロール別の細かい操作権限
- DB永続化
- レコード単位の差分マージ
- 監査ログの改ざん防止
- サーバーのHTTPS化と運用監視
